# Design Doc: Version 1.0.0 Rollout

**Date:** 2026-03-07 (America/Chicago)  
**Status:** Approved  
**Scope:** Set the project version to `1.0.0` across package metadata and display it on the CLI welcome screen.

## Goals

- Set the release version to `1.0.0` everywhere relevant.
- Keep Cargo crate versions and Arch package metadata aligned.
- Show the version on the CLI welcome screen without introducing a second version source.

## Non-Goals

- Changing runtime behavior beyond showing the version on welcome.
- Adding a separate version command or banner system.
- Updating unrelated docs unless they explicitly encode the old version.

## Chosen Approach

Update the Cargo manifests and `PKGBUILD` to `1.0.0`, then use Cargo’s compile-time package version in the CLI welcome screen via `env!("CARGO_PKG_VERSION")`.

## Implementation Notes

- Update:
  - `cli/Cargo.toml`
  - `gtk/Cargo.toml`
  - `crates/installer-core/Cargo.toml`
  - `PKGBUILD`
- Update the welcome screen in `cli/src/wizard.rs` to display the version string from Cargo metadata.

## Testing

- Run `cargo test --workspace`
- Run `cargo run -p access-os-installer-cli -- --help`
- Sanity-check the manifests and `PKGBUILD` show `1.0.0`
